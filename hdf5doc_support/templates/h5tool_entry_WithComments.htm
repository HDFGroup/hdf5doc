Tool Entry Template

A Template with Comments for HDF5 Tool Entries




Introduction

The purpose of this file is to describe the information that should be included in an HDF5 tool entry. If you have questions about writing a tool entry, you can use this file as a template.

In this file, you will find an element such as Tool Name or Syntax and a short instruction. After each instruction, more information on the element will appear in a light blue comment block. Light green comment blocks such as this one have general comments that describe how to use this file.

Start with the directions in the green comment block below.


Directions

  • Copy this file, and rename the copied file as tool_name.htm where tool_name is the name of the tool.
  • Edit the new file. In each of the sections shown below, replace the instruction sentence with the appropriate information.
  • When you are finished, remove the comment blocks, and then send a copy to the Doc Team for final editing and publishing.

A helper script is a command-line tool that can help you compile an application. There are helper scripts for the compiling of applications written in C, C++, and Fortran in both serial and parallel environments. The entries that describe these helper scripts follow the pattern of other tool entries with a few minor variations. An example of an HDF5 tool helper script is h5cc_and_h5pcc_Example.htm.


Variables

In some of the descriptions, a variable has been used instead of a specific item. These variables are generic phrases and are enclosed with angle quotation marks. One of the variables is «h5tool_name». Replace the variable with the name of the tool.


Tool Entry Files

Each tool entry is kept in a separate file. When the HDF5 Reference Manual is published, the files are combined. To help the combining process, we use a .htm extension on the file names, and we do not include the <html> and <body> tags at the beginning and end of each file.


Word vs. HTML

We would rather you used HTML. If you are not comfortable using HTML, use Word.

Syntax:
Insert the command-line syntax here.

Syntax

In the Syntax section, we specify the command-line syntax for the tool.

HDF5 tools use the following general pattern:

h5tool_name [OPTIONS] first_parameter second_parameter last_parameter

where

h5tool_name is the name of the tool,

OPTIONS are the available command-line arguments,

and the parameters are the available parameters.

The OPTIONS are the actions that the tool will take when the command-line is executed.

The parameters are usually the file(s) or object(s) that the tool will use for input or output or that the tool will act on.


Syntax Examples

The table below shows some specific examples.

Tool Command-Line Syntax
h5stat h5stat [OPTIONS] file
h5repack h5repack [OPTIONS] in_file out_file
h5import h5import infile in_options [infile in_options ...] -outfile outfile



Purpose:
Describe in this section what the tool accomplishes in a one-line statement that begins with a verb.

Purpose

In the Purpose section, we state what the function accomplishes when it is used.

Like a dictionary definition, we are looking for a clear and concise statement.


Writing a Purpose Statement

The following are some qualities of a good purpose statement.

  • Start with a verb. We write the purpose statement as a sentence that starts with a verb. A more complete sentence might start with a subject, but for purpose statements, the subject would always be “This tool ....” Instead of “This tool gets/retrieves/sets/writes ....”, we cut out the subject and are left with “Gets/Retrieves/Sets/Writes ....”

    Preferred verbs include the following:

    • Creates
    • Converts
    • Compares
    • Tests

  • Add useful details while keeping the statement short. In many cases, a bit of elaboration at the end of the sentence will help. For example, the first line in each pair below is more informative than the second.

    1. Converts a GIF file to an HDF5 file.
    2. Converts a GIF file.

    1. Creates a dataset at a specified location.
    2. Creates a dataset.

    1. Helper script to compile HDF5 C++ applications.
    2. Helper script to compile HDF5 applications.

Example Purpose Statements

The table below shows some specific examples.

Tool Purpose Statement
h5stat Reports statistics about an HDF5 file and its objects.
h5repack Copies an HDF5 file to a new file with or without compression and/or chunking.
h5import Imports data into an existing or new HDF5 file.
h5check Verifies that an HDF5 file is encoded according to the HDF5 specification.



Motivation:
Tell the reader in this section why they might want to use this tool.

Motivation

A Motivation section can help a user understand better why the tool might be used, can help a user understand the types of problems the tool addresses, or can help a user choose to use one tool instead of another. This is the section where the use cases can be described.


Writing a Motivation Statement

The following is a general way of writing a tool’s motivation.

«h5tool_name» is used in these circumstances to achieve these things and/or to address these problems.

Keep the motivation statement concise and high-level, but use multiple paragraphs if needed.

See the examples below.


Example Motivations

The table below shows some specific examples.

Tool Motivation Statement
h5check

h5check is a validation tool designed to verify that an HDF5 file is encoded according to the HDF5 File Format Specification. The purpose is to ensure data model integrity and long-term compatibility between evolving versions of the HDF5 Library.

Independent Verification Tool:

Note that h5check is designed to operate independently of the HDF5 Library:

  • It verifies the validity of an HDF5 file directly against the HDF5 File Format Specification without reference to or any use of the HDF5 Library.
  • h5check is distributed separately; see “ HDF5 Tools and Software.”
h5fix_swapped_ids

h5fix_swapped_ids corrects a metadata corruption error that manifests as incorrect encoding of fractal heap identifiers for attributes and shared object header messages. This error occurs only in particular circumstances and causes some files created on a big-endian system to be unreadable on a little-endian system.

h5fix_swapped_ids corrects the incorrect encoding.

This error existed in HDF5 Releases 1.8.0 through 1.8.4; it was fixed in HDF5 Release 1.8.4-patch1.

Comparing Purpose and Motivation

The aim of a purpose statement is to describe what a tool will acomplish when it is used. The aim of a motivation statement is to describe why a tool might be used. A purpose statement should be short. A motivation statement will usually be longer than a purpose statement. To illustrate the differences, the purpose and motivation statements for h5check and h5fix_swapped_ids are shown below.

Description:
In this section, provide a more detailed description of the tool and instructions for its use.

Description

In the Description section, we explain in more detail how the tool works and how to use the tool. The tool’s parameters can be briefly mentioned.

In general, parameters should be discussed in the “Parameters and Options:” section below. “Parameters and Options:” will include their complete descriptions and possible values. Important groupings of or relationships among parameters and interactions among parameters can be discussed either here or in “Parameters and Options:”; the determining factor is ease of user understanding.

Things that may not be obvious but that are important for correct use of the function should be discussed here. These include observations, caveats, special instructions, and warnings. The reader should never have to look at the source code or tests to figure these things out.


Writing a Description

Start the text in the Description section in the following pattern:

«tool_name» «verb» «what is compared, created, or verified»

where

«tool_name» is the name of the tool,

«verb» is the action the tool does,

Commonly used verbs include creates, converts, copies, updates, verifies, dumps, compares, tests, and prints.

and «what is compared, created, or verified» is the rest of the first sentence and describes what is compared, created, or verified.

The other sentences and paragraphs in the section will expand upon this first sentence.

See the examples below.


Description Examples

Some sample descriptions are included below. The tool name is followed by its description.

Tool Description
h5ls

h5ls prints selected information about specified HDF5 file(s) and/or object(s) in the specified format. In some cases, information regarding symbolic links is also provided.

h5copy

h5copy  copies an HDF5 object (a dataset, committed (named) datatype, or group) from an input HDF5 file to an output HDF5 file.

If a group is specified as the input object, any objects in that group will be recursively copied.

The output file may or may not already exist.

h5copy will fail if the destination object name already exists.

Notes and Warnings

If there is a risk of data corruption, silent failure, or unintended consequences that might go unnoticed, consider adding a Note: or Warning: lead-in to a paragraph. In general, such notes should come at the end of the “Description:” section.

Options:
List and describe the command-line options in this sub-section.

Options

We use options to control how the tool works. With various options, we can specify the action that the tool might take.

Options can have short and long formats. The short format is a dash and a single letter. The long format has two dashes and a word. The table below shows how to display the available options via some general cases.

Short Long Comments
-«s»   Short options use a dash with a single character.
-«s» «variable»   Short options may also have variables.
  --«long» Long options use two dashes and a word. Multiple words can be linked with underscores.
  --«long» «variable» Long options may also have variables.

Option Examples

Some examples of command-line options are shown in the table below

Short Long Comments
-h --help This commonly used option will tell the tool to print a usage message and then exit.
-v --verbose This commonly used option will tell the tool to print verbose output.
-l --long_option This option will list the short forms of the tool’s options and the long forms if there are any.
-G --groupmetadata This option is used with h5stat and will retrieve file space information for group metadata.
-S «fs_strategy» --fs_strategy= «fs_strategy» This option is used with h5repack and will specify the file space management strategy that will be used for the output file.

See the entries for the h5repack and h5stat tools for more examples of how options might be used. Note the variety of options.

Environment Variables Example

The “Environment Variables” section for the h5cc and h5pcc tools are shown below as an example.

When set, these environment variables override some of the built-in h5cc and h5pcc defaults.

HDF5_CC Use a different C compiler.
HDF5_CLINKER Use a different linker.
HDF5_USE_SHLIB=[yes|no]     Use shared version of the HDF5 library [default: no].
HDF5_CPPFLAGS Use additional preprocessor flags.
HDF5_CFLAGS Use additional C compiler flags.
HDF5_LDFLAGS Use additional library paths.

The last four of these environment variables have corresponding variables with names ending in BASE that can also be set by editing their values in the “Things You Can Modify to Override HDF5 Library Build Components” section of the h5cc and h5pcc scripts.

Note that adding library paths to HDF5_LDFLAGS where another HDF5 version is located may link your program with that other HDF5 Library version.




Example Usage:
Insert in this section examples that show how the tool is used.

Example Usage

In the Example Usage section, we include some examples of the tool being used. Include comments that describe what the options and parameters mean where appropriate. We usually put the description of what the example does before the example command-line.

One of the examples should show the tool’s default behavior. The default behavior of a tool is usually what happens when the tool is run without specifying any command-line options.

The following is the Example Usage section for h5repack:

Applies GZIP compression to all objects in file1 and saves the output in file2. Prints verbose output.

    h5repack -f GZIP=1 -v file1 file2

Applies SZIP compression only to object dset1.

    h5repack -f dset1:SZIP=8,NN file1 file2

Applies chunked layout to objects dset1 and dset2.

    h5repack -l dset1,dset2:CHUNK=20x10 file1 file2

The following is the Example Usage section for h5copy:

In verbose mode, create a new file, test1.out.h5, containing the object array in the root group, copied from the existing file test1.h5 and object array. 

    h5copy -v -i "test1.h5" -o "test1.out.h5" -s "/array" -d "/array

In verbose mode and using the flag shallow to prevent recursion in the file hierarchy, create a new file, test1.out.h5, containing the object array in the root group, copied from the existing file test1.h5 and object array. 

    h5copy -v -f shallow -i "test1.h5" -s "/array" -o test1.out.h5" -d "/array"

See the entries for these tools for more examples: h5dump, h5diff, h5mkgrp, and h5fix_swapped_ids.
