Name: H5Odecr_refcount \
Signature:\
herr_t H5Idecr_refcount(hid_t object_id)\ \
Purpose:\
Decrements the hard link count for an object.\
Description:\
H5Odecr_refcount decrements the hard link count of the object\ identified by object_id. This function should only be used by advanced users!\ \

\ The hard link count for an object is its reference count within the HDF5 file. This reference count\ is incremented every time a hard link to the object is created, and decremented every time a hard\ link to the object is unlinked. When this count reaches zero, the object will be deleted.\ \

\ This function and its counterpart H5Oincr_refcount can be used to emulate creating a hard\ link to an object. In general, this should only be done when user-defined links that reference the\ object by address are created and destroyed. User-defined links that reference this object by\ path should not alter the reference count, since they cannot access the object if their path to the\ object (a hard link) is unlinked.\

\ \ Only advanced users should create links to objects by address and alter hard link counts\ because discrepancies in the hard link count can result in file corruption. If the hard link\ count is incremented but never decremented and becomes greater than the actual number\ of hard links to the object then the object will never be deleted even if all links to it are\ unlinked, and that space within the HDF5 file will be inaccessible forever.\

\ If the link count is decremented without having been incremented first, the object may be\ deleted when there are still links pointing to its address. If this occurs, file corruption may\ result.\

Parameters:\
    \ \ \ \ \
    hid_t object_id    IN: Object identifier whose hard link count will be modified.
\
Returns:\
Returns a non-negative reference count of the object ID after\ decrementing it if successful; otherwise a negative value is returned.\
Fortran90 Interface: none currently\

Name: H5Oopen\ \
Signature:\
hid_t H5Oopen(hid_t obj_id,\ hid_tloc_id,\ const char * name\ hid_tlapl_id,\ \ )\
Purpose:
\
Opens an existing object of any type and returns an identifier for it.\
Description:\
H5Oopen returns an identifier for the object with the specified\ name at the specified location, loc_id. This object can be a\ group, a dataset, or a committed datatype.\

\ The location loc_id can be a group or file identifier.\

\ \ H5Oopen returns a group, dataset, or datatype identifier for the object\ opened. The type of the identifier returned can be determined using\ H5Iget_type, and the appropriate close\ function should be used to close it.\

\ The Link Access Property List lapl_id holds information\ that is used when traversing links along the path to the object.\

\ In general, it is preferable to use H5Gopen, H5Dopen,\ and H5Topen when the type of the object being opened\ is known. However, there are some cases, such as within a user-defined\ \pard\tx560\tx1120\tx1680\tx2240\tx2800\tx3360\tx3920\tx4480\tx5040\tx5600\tx6160\tx6720\tx7660\ql\qnatural\pardirnatural \cf0 link traversal function, when the type of the object being opened is not\ \pard\tx560\tx1120\tx1680\tx2240\tx2800\tx3360\tx3920\tx4480\tx5040\tx5600\tx6160\tx6720\ql\qnatural\pardirnatural \cf0 known and this function must be used instead.\

\

Parameters: \
    \ \ \ \ \ \ \ \ \ \ \ \
    hid_t loc_id    IN: File or group identifier within which object should be opened
    const char *nameIN: Name of object to open.
    hid_tlapl_idIN: Identifier for the link access property list to be used.

Name: H5Oopen_by_objno\ \
Signature:\
hid_t H5Oopen_by_objno(hid_t loc_id,\ unsigned longobjno[2]\ )\
Purpose:
\
Opens an existing object of any type by address and returns an identifier for it.\
Description:\
H5Oopen returns an identifier for the object with object number\ objno in the file specified by loc_id. This object can\ be a group, a dataset, or a committed datatype. This function should only be used by\ advanced HDF5 users due to the danger of file corruption.\

\ An object's object number is actually its address and can be retrieved using\ H5Gget_objinfo(). The object number will not change unless the\ object is deleted; for this reason, it is wise to use H5Oincr_refcount\ if an object's object number is stored in a link. H5Odecr_refcount should\ be used once the link is destroyed.\

\ The location loc_id can be a group or file identifier. If a group, it\ should be in the same file as the object to be opened. If a file, it should be the file\ containing the object to be opened.\

\ H5Oopen returns a group, dataset, or datatype identifier for the object\ opened. The type of the identifier returned can be determined using\ H5Iget_type, and the appropriate close\ function should be used to close it.\

\ This function should be used with care. If the object number does not correspond\ to a valid object header in the file specified, HDF5 could behave unpredictably and\ possibly corrupt the file.

\

Parameters: \
    \ \ \ \ \ \ \ \ \
    hid_t loc_id    IN: File or group identifier for the file in which the object is located
    unsigned longobjno[2]IN: Object number of object as returned by H5Gget_objinfo
\
Returns: \
Returns a valid identifier if successful; otherwise returns a negative value.\
Fortran90 Interface:none currently\
