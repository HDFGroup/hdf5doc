Name:            H5Tget_native_type

Signature:

            hid_t H5Tget_native_type(hid_t type_id, H5T_direction_t direction)

Purpose:

            Returns the native type of a datatype.

Description:

This high-level function basically returns the native datatype(i.e. H5T_NATIVE_INT) given a datatype identifier.  It mainly facilitates the H5Dread function while users have to go through several steps to figure out the data type of dataset to be read into memory.  This function also handles each component data type of compound, enumerate, array, and variable-length datatype.

 

When the library comes down to the point to choose a native type for a primitive data type, there are mainly two predefined orders according to the size of the data type.  Each chooses the first data type matching the input data type. 

 

1.      ascending order (for integers and float numbers; signed or unsigned also included):

integers:

H5T_NATIVE_CHAR

H5T_NATIVE_SHORT

H5T_NATIVE_INT

H5T_NATIVE_LONG

H5T_NATIVE_LLONG

 

float numbers:

H5T_NATIVE_FLOAT

H5T_NATIVE_DOUBLE

H5T_NATIVE_LDOUBLE

2.      descending order:

integers:

H5T_NATIVE_LLONG

H5T_NATIVE_LONG

H5T_NATIVE_INT

H5T_NATIVE_SHORT

H5T_NATIVE_CHAR

 

float numbers:

H5T_NATIVE_LDOUBLE

H5T_NATIVE_DOUBLE

H5T_NATIVE_FLOAT

Parameters:

            hid_t type_id

                        Identifier of a datatype to query.

            H5T_direction_t direction

                        The order of native data type(mainly integer and float type) for the library

to choose according to the size of data type.  Valid values are H5T_DIR_DEFAULT, H5T_DIR_ASCEND, H5T_DIR_DESCEND.

Returns:

            Returns the native datatype if successful; negative otherwise.
