Name: H5Pset_coll_metadata_read

Signature:
herr_t H5Pset_coll_metadata_read( hid_t accpl_id, hbool_t is_collective )

Purpose:
Sets metadata read mode to collective or independent (default).

Description:
H5Pset_coll_metadata_read sets the metadata read mode in the access property list accpl. This property specifies whether the HDF5 Library should perform metadata reads collectively (TRUE) or independently (FALSE).

Examples of HDF5 operations that can potentially issue metadata reads include opening a dataset, datatype, or group; reading an attribute; or issuing a get info call such as getting information for a group (H5Gget_info).

When the property to access metadata collectively is set to true on a file access property list that is used in creating or opening a file, the HDF5 Library assumes that all metadata read operations issued on that file identifier are going to be issued collectively from all ranks. If an application does that but issues one metadata read operation independently, then corruption will be introduced in the metadata cache and HDF5 Library behavior will be undefined.

To avoid the above risk of corruption, users may wish to avoid setting the collective metadata I/O property on the file access property list, and individually set it on object access property lists (dataset, group, link, datatype, attribute access property lists) for certain API operations. This will indicate that only operations issued with those access property lists are going to be called collectively and other operations may be called independently.

Warning:
As noted above, corruption will be introduced into the metadata cache and HDF5 Library behavior will be undefined when both of the following conditions exist:
  • A file is created or opened with a file access property list in which the collective metadata I/O property is set to TRUE.
  • Any function is called that triggers a metadata read while the file remains open with that file access property list.
An approach that avoids this corruption risk is described above.

Parameters:
hid_t accpl_id   IN: File, group, dataset, datatype, link, or attribute access property list identifier
hbool_t is_collective   IN: Boolean value indicating whether metadata reads are collective (TRUE) or independent (FALSE)
Default mode: Independent (FALSE)

Returns:
Returns a non-negative value if successful; otherwise returns a negative value.

Fortran Interface:
None
