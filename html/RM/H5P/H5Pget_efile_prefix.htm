Name: H5Pget_efile_prefix

Signature:
ssize_t H5Pget_efile_prefix( hid_t dapl_id, const char *prefix, size_t size )

Purpose:
Retrieves the prefix for external raw data storage files.

Description:
H5Pget_efile_prefix retrieves the file system path prefix for locating external files associated with a dataset that uses external storage. This will be the value set with H5Pset_efile_prefix or the HDF5 Library’s default.

If no prefix is set in the environment variable HDF5_EXTFILE_PREFIX, the prefix returned by H5Pget_efile_prefix will be the one used by HDF5 to locate external storage.

The value of size is the size in bytes of the prefix, including the NULL terminator. If the size is unknown, a preliminary H5Pget_elink_prefix call with the pointer prefix set to NULL will return the size of the prefix without the NULL terminator. In the subsequent call to get the actual prefix, size must be set to that value plus one (1).

See H5Pset_efile_prefix for a more complete description of file location behavior.


Notes:
The buffer for the prefix string must be allocated by the caller. The appropriate size can be determined by calling the function with prefix set to NULL. The call will return the buffer size in bytes via the return value.

A prefix set in the HDF5_EXTFILE_PREFIX environment variable overrides any prefix set via H5Pset_efile_prefix. In such a situation, the prefix set in HDF5_EXTFILE_PREFIX will be used rather than any value returned by H5Pget_efile_prefix. H5Pget_efile_prefix, being a property function, examines only the property list; it is unaware of the environment variable and thus cannot return its setting.


Parameters:
hid_t dapl_id   IN: Dataset access property list identifier
const char *prefix   IN/OUT: Dataset external storage prefix in UTF-8 or ASCII
(Path and filename must be ASCII on Windows systems.)
size_t size   IN: Size of prefix buffer in bytes

Returns:
Returns the size of prefix and the prefix string will be stored in prefix if successful.
Otherwise returns a negative value and the contents of prefix will be undefined.

Fortran Interface:
None
