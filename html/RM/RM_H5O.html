H5O: Link Interface

Object API Functions

The Object interface functions manipulate objects and reference counts in an HDF5 file.

The C Interfaces:
  • H5Oopen
  • H5Oclose
      
  • H5Oopen_by_addr
      
  • H5Oincr_refcount
  • H5Odecr_refcount

Name: H5Oclose
Signature:
herr_t H5Oclose( hid_t object_id )

Purpose:
Closes an object.

Description:
H5Oclose closes an object in an HDF5 file. 

This function is the companion to H5Oopen. It is used to 
close open file object in an HDF5 file, including groups,
datasets, and named datatypes.  It has the same effect as 
calling H5Gclose, H5Dclose, or H5Tclose.

H5Oclose is NOT used to close identifiers of objects that 
are not file objects, such as property lists and dataspaces.

Parameters:
    hid_t object_id     IN: Object identifier
Returns:
Returns a non-negative value if successful; otherwise returns a negative value.

Fortran90 Interface:
None.

History:
Release     C
1.8.0 Function introduced in this release.
Name: H5Odecr_refcount
Signature:
herr_t H5Odecr_refcount( hid_t object_id )

Purpose:
Decrements an object reference count.

Description:
Warning! This function is extremely dangerous!
Improper use can lead to file corruption, inaccessible data, and other very bad things!

H5Odecr_refcount decrements the hard link reference count for an object. It should be used any time a user-defined link that reference an object by address is deleted, and only after H5Oincr_refcount has already been used.

Parameters:
    hid_t object_id     IN: Object identifier
Returns:
Returns the object's new reference count if successful; otherwise returns a negative value.

Fortran90 Interface:
None.

History:
Release     C
1.8.0 Function introduced in this release.
Name: H5Oincr_refcount
Signature:
herr_t H5Oincr_refcount( hid_t object_id )

Purpose:
Increments an object reference count.

Description:
Warning! This function is extremely dangerous!
Improper use can lead to file corruption, inaccessible data, and other very bad things!

H5Oincr_refcount increments the hard link reference count for an object. It should be used any time a user-defined link that references an object by address is added. When the link is deleted, H5Odecr_refcount should be used.

Parameters:
    hid_t object_id     IN: Object identifier
Returns:
Returns the object's new reference count if successful; otherwise returns a negative value.

Fortran90 Interface:
None.

Name: H5Oopen
Signature:
hid_t H5Oopen( hid_t loc_id, const char *name, hid_t lapl_id )

Purpose:
Opens an object using its location identifier and name.

Description:
H5Oopen opens an object in an HDF5 file. 

This function opens an object in the same way that H5Gopen,
H5Topen, and H5Dopen do. However, H5Oopen doesn't require
the type of object to be known beforehand. This can be
useful in user-defined links, for instance, when only a
path is known.

The opened object should be closed again with H5Oclose
or H5Gclose, H5Tclose, or H5Dclose.

Parameters:
    hid_t loc_id IN: Object location identifier
    const char *name     IN: Object name
    hid_t lapl_id IN: Access property list identifier for the link pointing to the object
Returns:
Returns an object identifier for the opened object if successful; otherwise returns a negative value.

Fortran90 Interface:
None.

History:
Release     C
1.8.0 Function introduced in this release.
Name: H5Oopen_by_addr
Signature:
hid_t H5Oopen_by_addr( hid_t loc_id, haddr_t addr )

Purpose:
Opens an object using its file address.

Description:
Warning! This function is extremely dangerous!
Improper use can lead to file corruption, inaccessible data, and other very bad things! 

H5Oopen_by_addr opens an object using its address within an 
HDF5 file, similar to an HDF5 hard link. The open object
is identical to an object opened with H5Oopen() and should 
be closed with H5Oclose() or a type-specific closing
function (such as H5Gclose() ).

This function is very dangerous if called on an invalid
address. For this reason, H5Oincr_refcount() should be
used to prevent HDF5 from deleting any object that is
referenced by address (e.g. by a user-defined link).
H5Odecr_refcount() should be used when the object is
no longer being referenced by address (e.g. when the UD link
is deleted).

The address of the HDF5 file on disk has no effect on
H5Oopen_by_addr(), nor does the use of any unusual file
drivers. The "address" is really the offset within the
HDF5 file, and HDF5's file drivers will transparently
map this to an address on disk for the filesystem.

Parameters:
    hid_t loc_id IN: Object location identifier
    haddr_t addr     IN: Object's address in the file
Returns:
Returns an object identifier for the opened object if successful; otherwise returns a negative value.

Fortran90 Interface:
None.

History:

1.8.0 Function introduced in this release.
