What’s New in HDF5 1.8.0-alpha3

June 2, 2006

 

Background:

            The HDF5 1.8.0 release will represent a major update in the HDF5 library and utilities.  We have attempted to provide new capabilities and improve performance while retaining compatibility with previous releases.

 

Backward and Forward API Compatibility:

            This release contains many new API routines to take advantage of the new features, but at the same time attempts to provide stability for applications by continuing to make existing API routines available and operate in a backwardly compatible manner, whenever possible.

 

Backward and Forward Format Compatibility:

            The HDF5 1.8.0 library will read all existing HDF5 files, from this or any prior release.  Although this release contains features that require additions and/or changes to the HDF5 file format, by default this release will write out files that conform to a “maximum compatibility” principle.  That is, files are written with the earliest version of the file format that describes the information, rather than always using the latest version possible.  This provides the best forward compatibility by allowing the maximum number of older versions of the library to read files produced with this release.

            If library features are used that require new file format features, or if the application requests that the library write out only the latest version of the file format, the files produced with this version of the library may not be readable by older versions of the HDF5 library.

 

Major New Features:

The 1.8.0 release is currently in alpha release, so this list may yet change. Below is the list of features that will become gradually available with the 1.8.0-alpha, 1.8.0-beta and 1.8.0 releases.

Three feature lists follow:

  1. Functions that are thought to be stable and are included in this 1.8.0-alpha release.
  2. Features that are included in this 1.8.0-alpha but may change; e.g., function names, function behavior, or parameter setup may change.
  3. Features that are not included in this 1.8.0-alpha but that are expected by the time of the 1.8.0-beta and in the final 1.8.0 release.

 

The following features are stable and available in HDF5 Release 1.8.0-alpha:

 

·      Collective Chunk I/O in Parallel – The library now attempts to use MPI collective mode when performing I/O on chunked datasets when using the parallel I/O file driver.

·      New Chunked Dataset Filters – These new I/O filters allow better compression of certain types of data:

o      N-Bit Filter – This filter compresses data which uses N-bit datatypes

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5P.html#Property-SetNbit

o      Scale+Offset Filter – This filter compresses scalar (integer and floating-point) datatypes which stay within a range.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5P.html#Property-SetScaleoffset

For descriptions in the HDF5 User’s Guide, see http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/UG/UG_frame17SpecialTopics.html

·      Transforms on Data Transfer – This feature allows arithmetic operations (add/subtract/multiply/divide) to be performed on data elements as they are being written to/read from a file.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5P.html#Property-SetDataTransform

Text-to-datatype and datatype-to-text conversions – This feature enables the creation of a datatype from a text definition of that datatype and the creation of a formal text definition from a datatype.  The text definition is in DDL format; DDL definitions of HDF5 datatypes can be found in the “DDL in BNF for HDF5” (http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/ddl.html).

H5LTtext_to_dtype() creates an HDF5 data type based on the text description and returns the data type identifier.  Given a datatype identifier, H5LTdtype_to_text() creates a DDL description of the datatype.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/text_to_dtype/text-dtype.htm

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5LT.html#H5Ltext_to_dtype

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5LT.html#H5Ldtype_to_text

·      Support for Integer-to-Floating-point Conversion – It is now possible for the HDF5 library to convert between integer and floating-point datatypes.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5T.html#Datatype-Convert

·      Revised Datatype Conversion Exception Handling – It is now possible for an application to have greater control over exceptional circumstances (range errors, etc.) during datatype conversion.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/dtype_conversion/Overflow.html 

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/dtype_conversion/Conversion.html

·      Serialize Datatypes and Dataspaces – A set of routines to serialize/deserialize HDF5 datatypes and dataspaces.  These routines allow datatype and dataspace information to be transmitted between processes or stored in non-HDF5 files.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/encodeID/encode.htm

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5T.html#Datatype-Encode

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5T.html#Datatype-Decode

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5S.html#Dataspace-Encode

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5S.html#Dataspace-Decode

·      “NULL” Dataspace – A new type of dataspace, which allows datasets without any elements to be described.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5S.html#Dataspace-Create

·      Extendible ID API – A new set of ID management routines has been added, which allow an application to use the HDF5 ID-to-object mapping routines.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5I.html

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/expandedIDs/expandedIDs.htm

·      Meta-data Caching

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/UG/UG_frame17SpecialTopics.html   (Select “Metadata Caching.”)

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5F.html
Five functions: H5Fget_mdc_config, H5Fget_mdc_hit_rate, H5Fget_mdc_size, H5Freset_mdc_hit_rate_stats, and H5Fset_mdc_config.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5P.html 
Two functions: H5Pset_mdc_config and H5Pget_mdc_config.

·      High-Level Fortran APIs – Fortran APIs have been added for H5Lite, H5Image and H5Table.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5LT.html

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5IM.html

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5TB.html

·      New High-Level APIs – A Packet Table API has been added to the high-level interfaces.  These routines are designed to allow variable-length records to be added to tables easily.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5PT.html

·      A Dimension Scales API has been added to the high-level interfaces.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/H5DimScales/H5dimscale_Specification_1_0.pdf

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/HL/RM_H5DS.html

·      Tool Improvements – One new tool has been added, and existing tools were enhanced:

o      h5stat, a new tool, is still in development.  It is described in the next section, with other features that may change.

o      Improved speed of h5dump – Performance improvements have been made to h5dump to speed it up when dealing with files that have large numbers of objects.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/Tools.html#Tools-Dump

·      Better UNIX/Linux Portability – This release now uses the latest GNU “auto” tools (autoconf, automake, and libtool) to provide much better portability between many machine and OS configurations.  Building the HDF5 distribution can now be performed in parallel (with the gmake “–j” flag), speeding up the process of building, testing and installing the HDF5 distribution.  Many other improvements have gone into the build infrastructure as well.

 

The following features are included in the 1.8.0-alpha release but may change before 1.8.0-beta:

 

·      Object Copying – This feature allows an object in one HDF5 file to be easily copied to another HDF5 file.  This is done at a low-level in the HDF5 file, allowing entire group hierarchies to be copied quickly, as well as compressed datasets to be copied without going through a decompression/compression cycle.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5G.html#Group-Copy
http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5G.html#Group-CreateExpand
http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5G.html#Property-SetCopyObject
http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5G.html#Property-SetCreateIntermediateGroup
And related functions.

·      Getting more information from H5Gget_objinfo – This routine was expanded to provide more information about the object being queried.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5G.html#Group-GetObjInfo

·      Extendible Error API – A new set of error API routines has been added, which allow an application to integrate its error reporting with the HDF5 library error stack.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/doc/RM/RM_H5E.html

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/errorAPI/ErrorAPI.html

·      FORTRAN API Wrapper Improvements – Several improvements were made to the FORTRAN build infrastructure, as well as adding support for previously missing and new API routines.

·      C++ API Wrapper Improvements – Several improvements were made to the C++ build infrastructure, as well as adding support for previously missing and new API routines.

·      Tool Improvements – Several new tools were added, as well as new features to existing tools:

o      Added h5stat tool – This tool allows HDF5 files to be analyzed in various ways to determine useful statistics about the objects in the file, such as the numbers of objects per group, the sizes of datasets, the amount of free space in the file, etc.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/h5stat/h5stat-2006-03-05.pdf 

 

 

The following features are not stable or are not implemented in 1.8.0-alpha. They will be available in 1.8.0-beta but may change between in 1.8.0-beta and 1.8.0 release:

 

·      Compact Groups – This feature allows groups containing only a few links to take up much less space in the file.

·      New Large Group Storage – The method of storing groups with many links has been updated to be faster and more scalable.

·      External Links – This feature allows links in a group to refer to objects in another file, and for the library to access those objects as if they are in the current file. Creation Order in Groups – Groups now allow an index on the time links are created, allowing iteration and lookup of links by creation time as well as alphanumeric by name.

·      Link API – This new set of API routines allows links in groups to be treated as first-class objects in the HDF5 data model.  The link API allows creation of temporary, “anonymous” objects in a file as well as more flexible operations on links in groups.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/linkEncodings/H5L_API.htm

·      Intermediate Group Creation – This feature allows intermediate groups that don’t exist yet to be created when creating an object in a file.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/H5Copy/Creating%20Missing%20Groups.pdf

·      Support for Unicode Character Set – The UTF-8 Unicode encoding is now supported for strings in datasets, the names of links and the names of attributes.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/Unicode/Unicode.html

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/linkEncodings/Character_Encoding.pdf

·      Stackable Virtual File Drivers – The lowest level in the library, the Virtual File Driver (VFD) routines, was revised to allow VFDs to be stacked easily.  This allows the “logging” VFD to be applied to any VFD, for example.

http://hdf.ncsa.uiuc.edu/HDF5/doc_1.8pre/supplements/stackvfd/Stackable_VFD.pdf
