Client Library Wishes to Unregister from HDF5 Error Stack:

Pre-conditions:
  • Client library has registered an error class ID and error information with HDF5 error API.
Scenario:
  1. Client library uses error class ID to indicate that all error information for that error class should be removed from the error API.
Notes:
  1. Alternatively, the client library can unregister individual major or minor error types one by one.
