Name: H5Fset_jnl_config
Signature:
herr_t H5Fset_jnl_config( hid_t file_id, H5AC_jnl_config_t *config_ptr )

Purpose:
Gets jounaling configuration for target file.

Description:
H5Fset_jnl_config attempts to configure journaling on the target file according to the configuration supplied in config_ptr.

Parameters:
hid_t file_id    IN: File identifier of the target file.
H5AC_jnl_config_t *config_ptr   IN: Pointer to the instance of H5AC_jnl_config_t containing the desired configuration.

The fields of H5AC_jnl_config_t struct are defined as follows:

int version    Integer field indicating the the version of the H5AC_jnl_config_t in use.

This field should be set to H5AC2__CURR_JNL_CONFIG_VER (defined in H5ACpublic.h) and must be initialized for an H5Fset_jnl_config or H5Pset_jnl_config call.

hbool_t enable_journaling   Boolean flag set to TRUE if journaling is enabled and to FALSE if journaling is disabled.
char journal_file_path[]   Full path of the file used to store the metadata journal. This field is only defined if enable_journaling is TRUE.

At present, the length of the journal file path is restricted to no more than H5AC2__MAX_JOURNAL_FILE_NAME_LEN.

hbool_t journal_recovered   Boolean flag indicating that the journaled file being opened was not closed correctly and that the file has been been recovered successfully.

Since this field is only meaningful on file open, it should always be set to FALSE in this context.

size_t jbrb_buf_size   The size of individual buffers in the journal buffer ring buffer.

This size should be chosen to be some multiple of the block size used by the file system on which the journal file will be written.

int jbrb_num_bufs   Number of buffers in the journal buffer ring buffer.

If synchronous I/O is used, one or two buffers is sufficient. If asynchronous I/O is used, the number of buffers should be sufficiently large that a write on buffer is likely to complete before that buffer is needed again.

hbool_t jbrb_use_aio   Boolean flag indicating the use of asynchronous I/O for journal entry writes. TRUE indicates that asynchronous I/O is in use or is to be used; FALSE indicates no asynchronous I/O.
hbool_t jbrb_human_readable   Boolean flag indicating whether the journal file is written or is to be written in human-readable form (TRUE). FALSE indicates a machine-readable journal file.

In general, this field should be set to FALSE, as the human-readable journal file is at least twice a large as the machine-readable version.

Returns:
Returns a non-negative value if successful; otherwise returns a negative value.
