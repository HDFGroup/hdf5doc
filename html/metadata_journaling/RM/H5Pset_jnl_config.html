Name: H5Pset_jnl_config
Signature:
herr_t H5Pset_jnl_config( hid_t fapl_id, const H5AC_jnl_config_t *config_ptr )

Purpose:
Sets initial metadata journaling configuration.

Motivation:
This function is part of HDF5’s metadata journaling feature, which guards against metadata corruption and provides a recovery capability.

Metadata journaling is discussed more fully in “Using HDF5’s Metadata Journaling and File Recovery Features” and “Metadata Journaling to Improve Crash Survivability.”

Description:
H5Pset_jnl_config sets the initial metadata journaling configuration in the file access property list fapl_id according to the values supplied in the buffer specified by config_ptr.

This function is used when a file is to be opened with metadata journaling enabled.

Failure mode:
H5Pset_jnl_config will fail if an invalid configuration is detected.

Parameters:
hid_t fapl_id      IN: File access property list identifier.
 
const H5AC_jnl_config_t *config_ptr
    IN: Pointer to the instance of the H5AC_jnl_config_t struct containing the desired configuration.

See H5AC_jnl_config_t for the struct definition and a field-by-field description.

The fields of the struct should be set as follows:

  • Initialize version to H5AC2__CURR_JNL_CONFIG_VER, a value set internally by the HDF5 Library.
  • Set enable_journaling to TRUE if journaling is to be turned on, to FALSE if journaling is to be turned off.
  • Set journal_file_path to the full path of the file used to store the metadata journal. This path is defined only if enable_journaling is set to TRUE and can be no more than H5AC2__MAX_JOURNAL_FILE_NAME_LEN characters long.
  • Set journal_recovered to FALSE.
  • jbrb_buf_size specifies the size of each individual buffer in the journal buffer ring buffer. The specified value should be a multiple of the block size used by the file system on which the journal file will be written.
  • jbrb_num_bufs specifies the number of buffers in the journal buffer ring buffer. If synchronous I/O is used, one or two buffers will be sufficient. If asynchronous I/O is used, the number of buffers should be large enough that a write to a buffer is likely to complete before that buffer is needed again.
  • Set jbrb_use_aio to FALSE if the journal file should be written using synchronous I/O. If the host system supports asynchronous I/O, consider setting the field to TRUE to allow processing to overlap in time with journal file writes.
  • Set jbrb_human_readable to TRUE if the journal file should be human readable. Otherwise, use FALSE to create a more efficient machine-readable file.

Returns:
Returns a non-negative value if successful; otherwise returns a negative value.
