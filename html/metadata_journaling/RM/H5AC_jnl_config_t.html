Name: H5AC_jnl_config_t

Definition:
typedef struct H5AC2_jnl_config_t {
     /* struct version field: */
     int      version;
 
     /* metadata journaling configuration fields: */
     hbool_t  enable_journaling;
     char     journal_file_path[H5AC2__MAX_JOURNAL_FILE_NAME_LEN + 1];
     hbool_t  journal_recovered;
     size_t   jbrb_buf_size;
     int      jbrb_num_bufs;
     hbool_t  jbrb_use_aio;
     hbool_t  jbrb_human_readable;
} H5AC2_jnl_config_t;

Purpose:
Metadata journaling configuration data structure

Motivation:
This data structure is part of HDF5’s metadata journaling feature, which guards against metadata corruption and provides a recovery capability.

Metadata journaling is discussed more fully in “Using HDF5’s Metadata Journaling and File Recovery Features” and “Metadata Journaling to Improve Crash Survivability.”

Used By:
This struct is used by the following functions:
         H5Fset_jnl_config
         H5Fget_jnl_config
         H5Pset_jnl_config
         H5Pget_jnl_config

Field Description:
The H5AC_jnl_config_t fields are defined as follows:

int version
  • Integer field indicating the the version of H5AC_jnl_config_t in use
  • Always initialized by calling function.
hbool_t enable_journaling
  • Boolean flag
  • TRUE if journaling is enabled;
    FALSE if journaling is disabled.
char journal_file_path[]
  • Full path of the file used to store the metadata journal
  • Restricted in length to no more than H5AC2__MAX_JOURNAL_FILE_NAME_LEN characters.
  • Defined only if enable_journaling is TRUE.
hbool_t journal_recovered
  • Boolean flag
  • This field is meaningful only on file open following a recovery tool’s attempt to recover corrupted metadata in that file.
  • For use only by file recovery tools; should be set to FALSE in all other situations.
  • TRUE if journaled file being opened was not closed correctly but has been successfully recovered;
    FALSE under all other circumstances.
  • Unless writing a new recovery tool, a user will never set this flag to TRUE; it should always be FALSE unless the recovery tool has tried and been unable to recover the file.
size_t jbrb_buf_size
  • Size of individual buffers in the journal buffer ring buffer
  • Should be a multiple of the block size of the underlying file system.
int jbrb_num_bufs
  • Number of buffers in the journal buffer ring buffer
  • If synchronous I/O is used, one or two buffers will be sufficient. If asynchronous I/O is used, the number of buffers should be large enough that a write to a buffer is likely to complete before that buffer is needed again.
hbool_t jbrb_use_aio
  • Boolean flag
  • TRUE if asynchronous I/O is to be used for journal entry writes;
    FALSE if synchronous I/O is to be used for journal entry writes.
hbool_t jbrb_human_readable
  • Boolean flag
  • TRUE if journal file is written or is to be written in human-readable form;
    FALSE indicates a machine-readable journal file.
  • Unless there is specific reason to do otherwise, set this field to FALSE; a human-readable journal file will be at least twice as large as a machine-readable version.
