Documentation for Metadata Journaling and File Recovery

The metadata journaling and file recovery feature set is provided to guard against metadata corruption and to enable recovery of a file’s metedata in the event of corruption.

This feature set is documented as follows.

  • Design and user documents, with overviews of the feature set and its use:
    • “Using HDF5’s Metadata Journaling and File Recovery Features”
    • “Metadata Journaling to Improve Crash Survivability” (adapted from the feature RFC)

  • Reference manual entries for the HDF5 Library functions and data structure that govern metadata journaling:
    • H5Fset_jnl_config
    • H5Fget_jnl_config
    • H5Pset_jnl_config
    • H5Pget_jnl_config

    • H5AC_jnl_config_t

  • In some environments, events that result in corrupted metadata are inevitable. On such occasions, h5recover, a scriptable command-line tool can often rebuild the metadata from the journal record:
    • h5recover
Note that this feature set preserves only metadata; different steps must be taken to guard against raw data corruption.
